Типографика ST-style — кейсы и поведение

Все размеры ниже управляются только атрибутами фреймворка. Меняй ширину окна — в шапке видно активный брейкпоинт, размер --font-size и вычисленный размер образцового h1. Блоки с красной рамкой — container-query контексты.

Модель: --font-size — база (fluid-clamp() ramp по якорям снаружи / калиброванный cqw внутри query); --fs-scale — групповой множитель [fs] (наследуется); --fs = --font-size × --fs-scale; токены --fs-h1 = calc(3.2 × --fs) и т.д.

1 · Глобальный ramp --font-size (fluid clamp)

Якоря на каждом брейкпоинте (--font-size-xs…--font-size-xl, по умолчанию 10px + 0.75px × n), между ними — плавная интерполяция через clamp() (без скачков на границах). Вся типографика — множители --fs, поэтому растёт вместе. Тяни ширину окна — h1 в шапке меняется непрерывно, а не ступенями.

H2 — заголовок

p — базовый абзац. На мобиле ~12px, на десктопе ~18px. Ничего, кроме --font-size, менять не нужно: отступы, grid и контейнер не затронуты.

Отступы (m/p/gap) в ramp не участвуют — они на своей --space-step шкале и масштабируются отдельно (через container-query).

1b · Якоря ramp из значений UI

Каждому брейкпоинту — своя переменная-якорь. Копируешь фактические размеры из макета и присваиваешь их на :root (ramp считается там). В самих точках значение точное, между ними — fluid.

/* project root.css */
:root {
  --font-size-base: 14px;   /* <360px  */
  --font-size-md:   16px;   /* ≥768px  — вставил из макета */
  --font-size-lg:   20px;   /* ≥1280px */
  --font-size-xl:   22px;   /* ≥1920px */
  /* незаданные (xs/sm/lp/dt) — донастроятся линейно из $fs-step */
}

Живой тумблер (ставит эти якоря на :root всей страницы):

Якоря задаются на :root (глобально) — это дизайн-решение уровня темы, а не поддерева. В самих брейкпоинтах значения точные; наклон между ними печётся из конфига ($font-sizes-bp / $fs-step). Для точной параллели внутри query задавай якоря в конфиге — там берётся cqw-калибровка.

Ставит font-size из токена --fs-* (с !important) на любой тег — семантика не меняется.

3 · Кастомные токены размера

Токены — безразмерные множители --fs (как встроенный --fs-h1 = 3.2). Заданы в :root страницы: --fs-p1 = calc(1.1 × var(--fs-p)), --fs-lead = calc(1.5 × var(--fs-p)), --fs-hero = 4.5. Применяются через tx + --tx-mult и вливаются в ramp/scale/cqw (потому что реальный font-size = множитель × --fs считается на самом элементе).

Hero (--fs-hero)

Лид-абзац (--fs-lead) — крупнее обычного p в 1.5 раза.

Слегка увеличенный p1 (--fs-p1, ×1.1).

Для частых токенов добавь ключ в карту $font-sizes конфига — появится tx="hero". Рантайм-варианты: tx style="--tx-mult: N" или напрямую style="font-size: calc(N * var(--fs))".

5 · [fs] по брейкпоинтам

Mobile-first, больший брейкпоинт выигрывает. Тяни ширину — множитель меняется.

fs="d2 lg-4"

×0.90 на xs–lp, затем ×1.20 с lg (≥1280px).

fs="4 md-0"

×1.20 на xs–sm, затем ×1.00 с md (≥768px).

fs="0 sm-2 lg-4 xl-d2"

×1.00 → ×1.10 (sm) → ×1.20 (lg) → ×0.90 (xl).

6 · [fs]: обёртка vs. сам элемент

На элементе с собственным размером (тег p/h* или tx) множитель сохраняет базовый размер и домножает его. На «голом» div/span берётся базовый размер шрифта × множитель.

<p fs="4">

p сохраняет свой ×1.2 и ×1.20 сверху → крупный абзац

<div fs="4"> (голый)
div: базовый размер × 1.20 (нет множителя p)
<h3 tx="h3" fs="4">

tx закрепляет h3, fs домножает → h3 × 1.20

Правило: fs удобнее ставить на контейнер-группу. Прямо на текстовый элемент — тоже работает и сохраняет его размер.

7 · container="query" — fluid-типографика

Внутри источника font-size становится cqw, калиброванным так, что при 100% ширине контейнера px = дизайн-режиму этого брейкпоинта. Два контейнера разной ширины на одном экране → разный размер шрифта (следит за своей шириной).

container="query", width: 320px

H1 fluid

p fluid — оба масштабируются от ширины 320px

container="query", width: 560px

H1 fluid

тот же h1, но крупнее — контейнер шире

Сузь окно так, чтобы контейнер стал уже своей «дизайн-ширины» — h1 плавно уменьшится (значение в скобках у h1 показывает вычисленный размер).

8 · container="query lg-static" — заморозка на lg

Fluid до lg, затем шрифт (и отступы) «замерзают» в px. Полезно, чтобы на широких экранах текст не рос бесконечно.

H1: fluid до 1280px, фикс px от lg

Раздвинь окно за 1280px — размер перестанет меняться.

9 · container="static md-query" — наоборот

Фикс px на мобиле, fluid от md (≥768px). Ниже md шрифт стабильный, выше — тянется за шириной.

H1: фикс до md, fluid от md

До 768px — стабильно; после — масштабируется с контейнером.

13 · [lc] — ограничение строк (line-clamp)

Обрезает текст многоточием; поддерживает брейкпоинты (0 = выключить).

lc="2"

Длинный текст, который будет обрезан после второй строки многоточием. Здесь много слов, чтобы гарантированно получить перенос и увидеть обрезку в действии на любой ширине колонки.

lc="3 lg-0"

3 строки на мобиле/планшете, а на lg (≥1280px) ограничение снимается и текст показывается целиком. Здесь тоже добавлено достаточно слов для нескольких строк переноса и наглядности.

15 · Новые reset-правила

text-wrap: balance (заголовки) — ровные строки

Достаточно длинный заголовок, чтобы увидеть балансировку переносов по строкам

text-wrap: pretty (p) — нет висячих строк

Абзац с включённым pretty: последняя строка не остаётся с одним «висячим» словом, а перенос распределяется аккуратнее по всему блоку текста.

:disabled → cursor: not-allowed

textarea: field-sizing: content (авто-рост) + resize: vertical

hyphens: auto (нужен lang на <html>)
