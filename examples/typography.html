Типографика — tx · fs · fw · lc

Все размеры управляются только атрибутами. Меняй ширину окна — в шапке видно активный брейкпоинт, --font-size и вычисленный размер образцового h1. Fluid-масштабирование от ширины контейнера вынесено в отдельное демо — container-query; отступы — в spacing.

Модель: --font-size — база (fluid-clamp() ramp по якорям); --fs-scale — групповой множитель [fs] (наследуется); --fs = --font-size × --fs-scale; токены --fs-h1 = calc(3.2 × --fs) и т.д.

1 · Глобальный ramp --font-size (fluid clamp)

Якоря на каждом брейкпоинте (--font-size-xs…--font-size-xl, по умолчанию 10px + 0.75px × n), между ними — плавная интерполяция через clamp() (без скачков). Вся типографика — множители --fs, поэтому растёт вместе. Тяни ширину окна — h1 в шапке меняется непрерывно, а не ступенями.

H1 — заголовок первого уровня

H2 — заголовок

H3 — заголовок

H4

H5
H6

p — базовый абзац. На мобиле ~12px, на десктопе ~18px.

<!-- ничего задавать не нужно: родные теги уже берут --fs-* -->
<h1>Заголовок</h1>
<p>Абзац</p>

Отступы (m/p/gap) в ramp не участвуют — они на своей --space-step шкале.

1b · Якоря ramp из значений макета

Копируешь фактические размеры из макета и присваиваешь их на :root (ramp считается там). В самих точках значение точное, между ними — fluid.

/* project root.css */
:root {
  --font-size-base: 14px;   /* <360px  */
  --font-size-md:   16px;   /* ≥768px  — вставил из макета */
  --font-size-lg:   20px;   /* ≥1280px */
  --font-size-xl:   22px;   /* ≥1920px */
  /* незаданные (xs/sm/lp/dt) — донастроятся линейно из $fs-step */
}

Якоря задаются на :root — это дизайн-решение уровня темы. В самих брейкпоинтах значения точные; наклон между ними печётся из конфига ($font-sizes-bp / $fs-step).

2 · tx — именованный размер на любом элементе

span tx="h1"
span tx="h3" · tx="h5" · tx="p"
div, но выглядит как H2 (tx="h2")
<p tx="h2">Крупный текст</p>
<span tx="h5">Маленький заголовок</span>
<div tx="p">Обычный текст</div>
<button tx="h4">Кнопка</button>

Ставит font-size из токена --fs-* (с !important) на любой тег — семантика не меняется. Значения h1…h6, p и кастомные ключи из $font-sizes.

3 · Кастомные токены размера

Hero (--fs-hero)

Лид-абзац (--fs-lead) — крупнее обычного p в 1.5 раза.

Слегка увеличенный p1 (--fs-p1, ×1.1).

/* объявляем безразмерные множители --fs */
:root { --fs-p1: calc(1.1 * var(--fs-p)); }

<!-- применяем через --tx-mult -->
<p tx style="--tx-mult: var(--fs-p1)">p1</p>
<!-- эквивалент напрямую -->
<p style="font-size: calc(var(--fs-p1) * var(--fs))">p1</p>

Токены — безразмерные множители --fs. Применяй через реальный font-size (… × var(--fs)) — тогда токен fluid. style="font-size: var(--fs-p1)" не сработает: это множитель, не длина. Для частых токенов — ключ в карту $font-sizes.

4 · fs — групповой множитель (знаковые шаги 5%)

fs="d4" → ×0.80

Заголовок

абзац

fs="0" → ×1.00

Заголовок

абзац

fs="2" → ×1.10

Заголовок

абзац

fs="4" → ×1.20

Заголовок

абзац

Наследование и переопределение:

section fs="4" (×1.20)

H3 в группе — ×1.20

p — тоже ×1.20 (унаследовал от section)

…а этот p: fs="d6" → ×0.70, переопределил родителя

вложенный fs="0" → сброс к ×1.00

<section fs="2">        <!-- вся группа ×1.10 -->
    <h2>заголовок ×1.10</h2>
    <p>текст ×1.10</p>
    <p fs="d1">этот абзац ×0.95 (переопределил)</p>
</section>

fs="N" → ×(1 + N·0.05); fs="dN" → ×(1 − N·0.05); fs="0" — нейтрально. Наследуется всем поддеревом через --fs-scale; потомок переопределяет (абсолютно, не накопительно).

5 · fs по брейкпоинтам

Mobile-first, больший брейкпоинт выигрывает. Тяни ширину — множитель меняется.

fs="d2 lg-4"

×0.90 на xs–lp, затем ×1.20 с lg (≥1280px).

fs="4 md-0"

×1.20 на xs–sm, затем ×1.00 с md (≥768px).

fs="0 sm-2 lg-4 xl-d2"

×1.00 → ×1.10 (sm) → ×1.20 (lg) → ×0.90 (xl).

<div fs="2 lg-4">   <!-- ×1.10 на xs–lp, ×1.20 на lg+ -->
<div fs="d1 md-0">  <!-- ×0.95 → ×1.00 с md -->

Доступно 1..10 вверх (N) и вниз (dN) для каждого брейкпоинта. Масштаб касается только шрифта (не отступов).

6 · fs: обёртка vs. сам элемент

<p fs="4">

p сохраняет свой ×1.2 и ×1.20 сверху → крупный абзац

<div fs="4"> (голый)
div: базовый размер × 1.20 (нет множителя p)
<h3 tx="h3" fs="4">

tx закрепляет h3, fs домножает → h3 × 1.20

<p fs="4">размер p × 1.20</p>         <!-- токен p сохраняется -->
<div fs="4">базовый размер × 1.20</div> <!-- голый div -->
<h3 tx="h3" fs="4">h3 × 1.20</h3>
Правило: fs удобнее ставить на контейнер-группу. На элементе с собственным размером (тег p/h* или tx) множитель сохраняет базовый размер; на «голом» div/span берётся базовый размер × множитель.

7 · fw — вес шрифта

thin (100)

light (300)

normal (400)

semibold (600)

bold (700)

black (900)

<h3 fw="light">Лёгкий заголовок</h3>
<span fw="extrabold">Жирный текст</span>
<p fw="normal">Обычный текст</p>

Устанавливает font-weight с !important на любой элемент. Значения: thin · extralight · light · normal · medium · semibold · bold · extrabold · black (100…900).

8 · lc — ограничение строк (line-clamp)

lc="2"

Длинный текст, который будет обрезан после второй строки многоточием. Здесь много слов, чтобы гарантированно получить перенос и увидеть обрезку в действии на любой ширине колонки.

lc="3 lg-0"

3 строки на мобиле/планшете, а на lg (≥1280px) ограничение снимается и текст показывается целиком. Здесь тоже добавлено достаточно слов для нескольких строк переноса и наглядности.

<p lc="2">обрезать после 2 строк</p>
<p lc="3 lg-0">3 строки, на lg+ ограничение снято (0)</p>
<p lc="4 md-0 lg-10">4 → выкл → 10</p>

Обрезает текст многоточием; поддерживает брейкпоинты. Числа 1–10, 0 = выключить.

9 · Reset-типографика (автоматически)

text-wrap: balance (заголовки) — ровные строки

Достаточно длинный заголовок, чтобы увидеть балансировку переносов по строкам

text-wrap: pretty (p) — нет висячих строк

Абзац с включённым pretty: последняя строка не остаётся с одним «висячим» словом, а перенос распределяется аккуратнее по всему блоку текста.

hyphens: auto (нужен lang на <html>)

Переносослияниепереносослияние — узкая колонка демонстрирует переносы по слогам.

<!-- применяется автоматически reset'ом, без атрибутов -->
<h2>...</h2>  /* text-wrap: balance */
<p>...</p>    /* text-wrap: pretty; hyphens: auto */
<html lang="ru"> <!-- нужен для hyphens -->

Эти правила из CSS Reset применяются к h1–h6 и p без атрибутов. Полный список reset — в документации.
